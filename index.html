Gödel.js

This is a loose, JavaScript implementation of the S language of Computability, Complexity, and Languages by Davis et al.

What is this?

The whole premise is that you can build arithmetic from just 3 simple operations: increment, decrement, and conditional jumps.

Of course, Davis defines a conditional jump with a test for equality. Hence, the set of core operations is more like: increment (incr), decrement (decr), conditional branch (cond), and equality (eq).

Caveats

Negative numbers

Interestingly enough, negative numbers are not supported in the algebra - thus making it toy.

Take for example, implementing lt (less than). For positive numbers, this is easy. Our definition of lt would state that x is less than y if (through repeated decrements) x hits zero before y. For negative numbers, you'd be able to just increment x or y and see if x hits zero first. However, to determine whether you should decrement or increment, you need to determine if either of the inputs are negative.

How do we determine if the inputs are negative? Well, a negative number is one that is less than zero... Hence, to implement lt for negative numbers, you'd need lt to determine if a number was negative.

This is why the algebra only supports positive numbers.

Floating point numbers

Floating point numbers would add a wealth of complexity to the algebra. Base operations like sub would need to account for the subtraction of decimals – requiring a more powerful decr and base primitives to determine if a number is a float.

This is why the algebra only supports integers.

The computational model, thus far:

Core operations:

incr: increment

incr(4) => 5

decr: decrement (stops at zero)

decr(1) => 0
decr(0) => 0

cond: conditional branch

cond(some_condition, then_callback_or_value, else_callback_or_value)

cond(lt(1, 2), function () {
    return true;
},
function () {
    return false;
})

// OR simply

cond(lt(1, 2), true, false)

eq: equality

eq(1, 1) => true

Supported arithmetic operations:

neq: not equal to

neq(2, 1) => true

lt: less than

lt(2, 1) => false

gt: greater than

gt(2, 1) => true

lte: less than or equal to

lte(1, 1) => true

gte: greater than or equal to

gte(1, 2) => false

add: addition

add(1, 3) => 4

sub: subtraction (stops at zero)

sub(2, 1) => 1
sub(2, 3) => 0 // no negatives

mult: multiplication

mult(2, 2) => 4

exp: exponentiation

exp(2, 2) => 4
exp(2, 0) => 1

Boolean operations:

or

or(lt(1, 2), gt(1, 2)) => true

and

and(lt(1, 2), gt(1, 2)) => false
and(lt(1, 2), gt(2, 1)) => true

not

not(lt(2, 1)) => true
